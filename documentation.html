Secure File Delete - Documentation

Introduction

The Secure File Delete application is designed to securely delete files from a specified folder. It provides various options for file selection and employs multiple steps and algorithms to ensure that the files are effectively and securely deleted, making them unrecoverable.

Architecture

The application is built using the following components:

  • Command-Line Interface (CLI): The application accepts command-line arguments to specify the folder path, file selection criteria, and other options.
  • File System Operations: The application interacts with the file system to retrieve the list of files, rename files, overwrite file content, and delete files.
  • Secure Deletion Algorithms: The application utilizes multiple secure deletion algorithms to ensure the files are securely wiped from the storage media.

Usage

The application can be used by executing the command safedelete followed by the desired parameters. The supported parameters are:

  • --folder, -f <folder_path>: Specifies the path of the folder to delete files from. This parameter is required.
  • --time_limit, -t <minutes>: Specifies the time limit (in minutes) for the maximum file age. Files older than the specified time limit will be selected for deletion. This parameter is optional.
  • --pattern, -p <file_pattern>: Specifies the file name pattern to match. Only files with names matching the specified pattern will be selected for deletion. This parameter is optional.
  • --exclude_files, -e <file1,file2,...>: Specifies a comma-separated list of files to exclude from deletion. These files will be skipped during the deletion process. This parameter is optional.
  • --recursive, -r: Enables recursive search for files in all subfolders of the specified folder. If this flag is not provided, only files in the top-level folder will be considered. This parameter is optional.
  • --help, -h: Displays the usage information for the application. This parameter is optional.

Workflow

The application follows the following workflow:

  1. Parses the command-line arguments and checks for required parameters.
  2. Maps short parameter aliases to their corresponding full parameter names.
  3. Performs secure file deletion based on the provided parameters.
  4. Retrieves the list of files in the specified folder that match the selection criteria.
  5. Displays the list of files to be securely deleted and prompts for confirmation.
  6. If confirmed, securely deletes the files using the following steps and algorithms:
    1. Renames the file multiple times with random names.
    2. Overwrites the file content with random data, null values, zero values, alternating patterns, encrypted random data, and null values again.
    3. Deletes the file by truncating its content and modifying its metadata.
  7. Displays the summary of the deletion process.

Examples

Here are some examples of using the Secure File Delete application:

  • safedelete --folder C:\Documents --time_limit 30 --pattern *.txt --exclude_files important.txt: Deletes all text files in the "C:\Documents" folder that are older than 30 minutes, excluding the file "important.txt".
  • safedelete --folder /home/user/Documents --pattern *.docx --recursive: Deletes all Word documents in the "/home/user/Documents" folder and its subfolders.
  • safedelete --folder D:\Photos --pattern * --exclude_files vacation.jpg,party.png: Deletes all files in the "D:\Photos" folder except for "vacation.jpg" and "party.png".

Security Considerations

It is important to note the following security considerations when using the Secure File Delete application:

  • The application permanently deletes files and makes them irrecoverable. Ensure that you have a backup of any important files before running the application.
  • The secure deletion process can be time-consuming, especially for large files or when using multiple overwrite passes. Consider the performance impact accordingly.
  • Exercise caution when specifying the folder path and file selection criteria to avoid unintended file deletion. Double-check the parameters before confirming the deletion.
  • The application does not provide a recovery mechanism for mistakenly deleted files. Take appropriate measures to prevent accidental data loss.

Conclusion

The Secure File Delete application provides a reliable and secure method to delete files from a specified folder. By utilizing multiple steps and algorithms, it ensures that the files are irreversibly wiped, making them unrecoverable. Use the application with caution and follow the security considerations to avoid any unintended data loss.
